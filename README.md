# FITNESS

Telegram-бот для фитнес-сопровождения: анкеты, диагностика, калькуляторы и оплата.

## Структура проекта

```text
app/
├── bot/
│   ├── handlers/        # Команды и сценарии бота (/start, «Обо мне», диагностика)
│   ├── keyboards/       # Кнопки и меню
│   └── states/          # FSM-состояния (диагностика, анкета, донат)
├── calculators/         # Формулы, расчеты и beta-модули
├── data/                # Контент: тренер, услуги, отзывы, товары
├── db/                  # Работа с БД
├── services/            # Интеграции (платежи, уведомления)
├── config.py            # Загрузка и валидация env-переменных
└── main.py              # Точка входа
```

## Требования и установка

### Требования

- Python 3.11+
- pip
- Telegram Bot Token
- Платежные токены (если используете оплату)

### Установка

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
pip install -r requirements.txt
cp .env.example .env
```

После этого заполните `.env` своими значениями.

## ENV-переменные

Обязательные переменные (см. `.env.example`):

- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота
- `PROVIDER_TOKEN` — токен платежного провайдера
- `SHOP_ID` — ID магазина
- `SECRET_KEY` — секретный ключ API
- `ADMIN_ID` — Telegram ID администратора (по умолчанию `5948629306`)
- `DATABASE_PATH` — путь к SQLite-файлу (по умолчанию `fitness.db`)

Если обязательные переменные не заполнены, бот не стартует и выдаст ошибку валидации конфигурации.

## Запуск локально

```bash
source .venv/bin/activate  # Windows: .venv\\Scripts\\activate
python -m app.main
```

## Запуск в Amvera

В проекте есть `amvera.yml` с настройками окружения. Базовый порядок:

1. Подключить репозиторий в Amvera.
2. В настройках сервиса добавить env-переменные из `.env.example` (см. раздел **ENV-переменные** выше).
3. Для SQLite на persistent mount задайте `DATABASE_PATH` в директорию постоянного тома Amvera (пример: `/data/fitness/fitness.db`).
4. Убедитесь, что директория в `DATABASE_PATH` доступна на запись приложению (бот создаст её автоматически при старте).
5. Актуальная точка входа приложения: `app/main.py`.
6. Команда запуска для Amvera: `python3 -m app.main`.
7. Запустить деплой и проверить логи старта.
8. Для автодеплоя в Amvera используйте ветку репозитория `main`.

Короткий чек-лист перед релизом:

- [ ] Env-переменные заполнены в настройках сервиса Amvera (ориентир — `.env.example` и раздел **ENV-переменные** в этом README).
- [ ] Для прода `DATABASE_PATH` указывает на persistent mount (например, `/data/fitness/fitness.db`).
- [ ] В команде запуска указано: `python3 -m app.main`.
- [ ] В логах после старта есть сообщения о запуске polling (например, `Start polling` / `Polling started`) и нет ошибок валидации конфигурации.

## Ручная проверка (smoke-test)

Проверки удобно делать из Telegram-клиента после запуска бота.

1. **Меню `/start`**
   - Отправить `/start`.
   - Проверить, что появляется главное меню и активны кнопки.

2. **Раздел «Обо мне»**
   - Нажать кнопку/пункт «Обо мне».
   - Проверить отображение профиля тренера, услуг и отзывов.

3. **Быстрая диагностика**
   - Запустить сценарий быстрой диагностики.
   - Пройти шаги, проверить вывод интерпретации и рекомендации.

4. **Полная анкета**
   - Пройти полную анкету целиком.
   - Проверить, что шаги не пропускаются, ответы сохраняются, итог формируется корректно.

5. **Донат / оплата**
   - Запустить сценарий доната/оплаты.
   - Проверить создание инвойса и обработку успешного платежа (в тестовом режиме провайдера).

6. **Проверка HTML-форматирования**
   - Откройте разделы «Обо мне» → «Профиль тренера», «Услуги», «Отзывы», «Контакты».
   - Убедитесь, что текст из тегов отображается форматированно (жирный), а не как сырой текст вида `<b>...</b>` или `<code>...</code>`.
